# Taskoria: клиентская часть

Клиентская часть Taskoria - мобильное приложение на Flutter для Android. Приложение помогает планировать и выполнять повседневные задачи через геймификацию: задачи становятся квестами, пользователь получает XP и Coins, развивает виртуальный город, открывает достижения и отслеживает личный прогресс.

Проект относится к приложению **"Геймифицированное мобильное приложение для управления задачами и визуализации прогресса - Taskoria"**.

## Технологический стек

## Архитектурные решения клиентской части

Клиентская часть разделена на несколько уровней:

- `app/` содержит точку сборки приложения, тему и маршрутизацию.
- `features/` содержит пользовательские экраны: город, задачи, календарь, магазин, профиль, достижения, авторизация, уведомления и настройки.
- `shared/models/` содержит модели предметной области: задача, подзадача, профиль, город, здание, достижение, товар магазина.
- `shared/providers/` связывает UI с источниками данных через Riverpod.
- `shared/network/` содержит HTTP-клиент для работы с backend.
- `shared/session/` управляет состоянием авторизации, выходом из аккаунта и локальным хранением токена.

Для данных используется Repository-подход. У каждого крупного раздела есть абстракция репозитория и две реализации: mock-реализация для локальной разработки и API-реализация для режима `core_api`.

## Изометрический город

Город строится в `lib/features/home/widgets/isometric_city_map.dart`.

Основная идея:

- карта состоит из изометрической сетки 28 x 28 условных ячеек;
- визуальная подложка собирается из блоков 2 x 2;
- дороги задаются явно как набор дорожных блоков;
- здания и декоративные объекты размещаются не по центру картинки, а по измеренной anchor-точке;
- для каждого здания указывается набор характеристик: категория, уровень, footprint на сетке, размер вывода, исходный размер изображения, anchor-точка изображения и целевая ячейка;
- функция `cellBottomCorner` переводит координаты ячейки в экранную точку изометрической карты;
- позиция изображения вычисляется так, чтобы выбранная anchor-точка изображения совпала с нижним углом нужной ячейки;
- порядок слоев считается по глубине: чем ниже объект на карте, тем позже он рисуется и тем выше оказывается визуально;
- для спорных наложений используются `layerDepthBias` и `layerPriority`.

Здания категорий связаны с игровыми категориями:

- учеба - школа;
- здоровье - госпиталь;
- работа - офис;
- личное - личное здание;
- ратуша - сводка профиля и прогресса.

У каждого здания может быть несколько уровней. На карте отображается уровень, который приходит из состояния города. Декоративные объекты из магазина отображаются только после покупки и размещения.

## Режимы данных

Приложение поддерживает два режима работы.

Mock-режим:

```powershell
flutter run --dart-define=USE_MOCK_DATA=true
```

Режим backend:

```powershell
flutter run --dart-define=APP_DATA_MODE=core_api --dart-define=API_BASE_URL=http://10.0.2.2:8000
```

Для физического Android-устройства вместо `10.0.2.2` нужно указать локальный IP-адрес компьютера, на котором запущен backend.

Если backend запущен в Docker, правило такое же: Docker работает на компьютере, а телефон подключается к опубликованному порту `8000` на компьютере.

Вариант для телефона в одной Wi-Fi сети с компьютером:

```powershell
flutter run --dart-define=APP_DATA_MODE=core_api --dart-define=API_BASE_URL=http://<PC_LOCAL_IP>:8000
```

Вариант для телефона по USB:

```powershell
adb reverse tcp:8000 tcp:8000
flutter run --dart-define=APP_DATA_MODE=core_api --dart-define=API_BASE_URL=http://127.0.0.1:8000
```
